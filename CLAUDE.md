# CLAUDE.md

Coding guidelines

Prefer object-oriented design.

Follow SOLID principles.

Keep changes scoped: do not modify more files than needed to complete the task.

Code must be testable.

Functions should do one thing, within reason.

Do not proactively modify existing files just to improve style.

All I/O, including databases and network access, must be behind interfaces.

## Project

nano-weather — a minimalist open source weather app for Android.
